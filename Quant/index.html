基础知识

ETL

提取（extract）、转换（transform）、加载（load），数据处理流程中的核心环节，本质是一套数据处理框架，通过三个连贯的步骤，实现从数据源到目标存储的高效数据流转。

PostgreSQL

https://www.runoob.com/postgresql/postgresql-tutorial.html

一个免费的对象-关系数据库服务器(ORDBMS)

Pandas

Pandas 是 Python 语言的一个扩展程序库，用于数据分析。

其名字衍生自术语 “panel data”（面板数据）和 “Python data analysis”（Python 数据分析）。

是一个开放源码、BSD 许可的库，提供高性能、易于使用的数据结构和数据分析工具。

是一个强大的分析结构化数据的工具集，基础是 Numpy（提供高性能的矩阵运算）。

Parquet

Apache Parquet是一种高效的列式存储格式，旨在提供在大数据处理环境中的高性能和可扩展性。它使用了一个自描述的、存储效率高的、高度可扩展的列式存储格式，以便有效地存储和处理大规模数据集。

  • 优势

在数据分析的日常工作中，我们常常需要处理 CSV、Parquet 等格式的文件，用 Pandas 做数据探索，或者快速搭建一个轻量的分析环境。但问题也随之而来：

Pandas 虽然灵活，但在处理大规模数据时显得吃力；
传统数据库（如 PostgreSQL、MySQL）需要独立安装、维护服务，部署复杂；
一些内存计算工具虽快，但缺少完整 SQL 支持，难以扩展。

DuckDB 作为“数据界的 SQLite”走进了越来越多开发者的视野。它不需要单独的服务进程，可以直接嵌入到 Python、R 或者应用程序中使用；同时支持 SQL 的完整能力，还能高效读取和写入常见文件格式。

FastAPI

FastAPI 是一个现代、快速（高性能）的 Web 框架，用于基于标准 Python 类型提示构建 API。

APScheduler

Advanced Python Scheduler

一个轻量级的 Python 定时任务调度框架。APScheduler 支持三种调度任务：固定时间间隔，固定时间点（日期），Linux 下的 Crontab 命令。同时，它还支持异步执行、后台执行调度任务。

React

React 是一个用于构建用户界面的 JAVASCRIPT 库，主要用于构建 UI

TypeScript

TypeScript 是 JavaScript 的一个超集，在 JavaScript 的基础上增加了静态类型检查的超集。
其设计目标是开发大型应用，它可以编译成纯 JavaScript，编译出来的 JavaScript 可以运行在任何浏览器上。

ECharts

ECharts 是一个使用 JavaScript 实现的开源可视化库，涵盖各行业图表，满足各种需求。

ECharts 提供了丰富的图表类型和交互能力，使用户能够通过简单的配置生成各种各样的图表，包括但不限于折线图、柱状图、散点图、饼图、雷达图、地图等。

JupyterLab

Jupyter源于Ipython Notebook，是使用Python（也有R、Julia、Node等其他语言的内核）进行代码演示、数据分析、可视化、教学的很好的工具，对Python的愈加流行和在AI领域的领导地位有很大的推动作用。

SQLAlchemy

SQL 工具包和对象关系映射器是一套全面的工具，用于处理数据库和 Python。它有几个不同的功能领域，可以单独使用或组合使用。

如果你做 ETF/股票/期货混合，资产模型要一开始分清楚：

Stock:

  • T+1
  • 涨跌停
  • 停牌
  • 手续费/印花税
  • 复权

ETF:

  • T+0 或 T+1，取决于市场和品种
  • 管理费可暂时忽略
  • 流动性和折溢价可后续处理

Future:

  • 保证金
  • 合约乘数
  • 到期换月
  • 夜盘
  • 逐日盯市
  • 多空都可开

最小可用版本 MVP

我建议第一版就做这些：

  1. 导入 ETF/股票/期货日线数据到 Parquet
  2. PostgreSQL 存标的信息和回测结果
  3. 实现 DataRepository
  4. 实现目标权重型 Strategy
  5. 实现日线 BacktestEngine
  6. 支持手续费和滑点
  7. 保存净值、交易、持仓、指标
  8. FastAPI 暴露回测接口
  9. React 展示回测结果

MVP 路径：

先只支持日线
先只支持收盘价调仓
先只支持目标权重策略
先只支持单账户
先把 ETF/股票跑通
再加期货保证金和换月

这个顺序比较稳。期货比股票/ETF复杂，尤其是合约换月、保证金、夜盘、主力连续合约，所以建议放在第二阶段。
