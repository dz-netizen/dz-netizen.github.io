cpp

阶段学习

增强版C

引用

  • 引用是变量的别名，定义时必须绑定到一个已有对象

  • 语法：type &name = obj;

  • 引用一旦绑定，不能再改绑到其他对象

  • 常用于函数形参，避免拷贝大对象

  • const 引用可以绑定到临时对象、字面量或表达式结果

string

  • std::string 是标准库字符串类，比 C 风格字符串更安全、更好用

  • 支持直接赋值、拼接、比较、查找、截取等操作

  • 常用接口：size()、empty()、substr()、find()、append()、c_str()

  • 读取整行文本通常用 getline(cin, s)，而不是 cin >> s

  • cin >> s 遇到空格会截断，getline 会读取整行

vector

  • std::vector 是动态数组，元素在内存中连续存放

  • 常用接口：push_back()、pop_back()、size()、empty()、operator[]、at()、begin()、end()

  • operator[] 不做边界检查，at() 会检查越界

  • 扩容时可能发生重新分配，因此不要长期保存失效的指针、引用和迭代器

  • 遍历时也可以使用范围 for

cin/cout

  • cin 和 cout 是标准输入输出流

  • >> 用于格式化输入，<< 用于格式化输出

  • endl 会换行并刷新缓冲区，普通情况下更常用 \n

  • 可以关闭同步提升输入输出速度

  • 输入失败时可以通过流状态判断，例如 if (!cin)

auto

  • auto 让编译器根据初始化表达式推导类型

  • 适合类型很长、模板迭代器复杂、范围 for 等场景

  • auto 需要初始化，不能单独声明

  • auto 会丢掉顶层 const 和引用修饰，若要保留，需要显式写 auto & 或 const auto &

  • 常配合范围 for、lambda、迭代器一起使用

STL

  • STL 是标准模板库，核心思想是容器、算法、迭代器分离

  • 容器负责存数据，算法负责处理数据，迭代器负责连接二者

容器

  • 顺序容器保存“线性序列”数据，常见有 vector、deque、list、string

  • 关联容器保存“按键组织”的数据，常见有 set、map、multiset、multimap

  • 无序关联容器基于哈希表，常见有 unordered_set、unordered_map

  • 容器适合场景大致如下：

    • vector：随机访问快，尾部插入删除快，最常用
    • deque：两端插入删除方便
    • list：任意位置插入删除稳定，但随机访问差
    • set/map：自动有序，适合查找、去重、范围查询
    • unordered_set/map：平均查找快，适合只关心存在性和计数

  • size() 返回元素个数，empty() 判断是否为空

  • push_back() 常用于 vector、string、deque

  • insert() 常用于 set、map、unordered_*

  • erase() 删除元素时要注意返回值和迭代器失效问题

迭代器

  • 迭代器可以理解为“泛化指针”，用于统一访问不同容器

  • 常见迭代器操作有自增、解引用、比较

  • 迭代器配合算法使用，是 STL 的核心用法

  • 范围 for 本质上也是通过迭代器实现的

  • begin() 指向首元素，end() 指向尾后位置，不能解引用

  • 对关联容器而言，迭代器遍历顺序是按键排序后的顺序

算法

  • 算法通常不直接操作容器，而是操作迭代器区间 [first, last)

  • 常见算法：sort、find、count、reverse、lower_bound、upper_bound、accumulate

  • 需要头文件 <algorithm>、<numeric> 等

  • sort 只适用于随机访问迭代器，所以不能直接对 list 排序

  • 需要自定义比较器时，可以传入函数对象、函数指针或 lambda

  • lower_bound 用于找第一个不小于目标值的位置，常用于有序数组二分查找

容器适配器

  • 容器适配器是对底层容器的封装，常见有 stack、queue、priority_queue

  • stack 是后进先出，常用接口：push()、pop()、top()

  • queue 是先进先出，常用接口：push()、pop()、front()、back()

  • priority_queue 是优先队列，默认每次取最大值

  • 优先队列常用于“每次取当前最大/最小元素”的问题

常用技巧

  • 排序后可以结合 unique 去重
  • pair 常用于存放二元组，make_pair 可以快速构造

  • map 和 unordered_map 都支持通过 operator[] 访问，但 operator[] 可能会创建新元素

  • 需要遍历键值对时可以使用范围 for

  • 选择容器时优先从需求出发，不要只看“能不能用”，要看“是否适合”

类和对象

  • 类是对现实对象的抽象，包含属性和行为

  • 对象是类的实例

  • 访问控制：public、private、protected

  • 封装的目标是隐藏内部实现，只暴露必要接口

  • 构造函数负责初始化对象，析构函数负责资源释放

  • this 指针指向当前对象

  • 成员函数加 const 表示不会修改对象状态

  • 友元可以突破访问控制，但应尽量少用

封装

  • 把数据和操作数据的函数放在一起，外部通过接口访问

  • 通过 private 保护内部状态，避免对象被随意破坏

继承

  • 继承描述“is-a”关系，子类可以复用父类接口和实现

  • 基类析构函数如果要通过基类指针删除派生类对象，通常应声明为虚函数

  • 继承方式有 public、protected、private

多态

  • 多态允许通过基类指针或引用调用派生类实现

  • 运行时多态依赖虚函数

  • 重写虚函数时建议加 override

  • 基类接口尽量设计成稳定、抽象的行为

现代cpp

  • 现代 C++ 强调资源管理、类型安全、表达能力和泛型编程

  • 常见特性：nullptr、constexpr、enum class、override、final、lambda、move、range for、智能指针

  • 用 nullptr 替代 NULL

  • 用 enum class 避免枚举污染命名空间

  • 用智能指针管理动态资源，减少手动 new/delete

  • 移动语义用于高效转移资源所有权，减少不必要拷贝

  • 右值引用 && 常与 std::move 一起出现

  • Lambda 适合就地定义短小回调和比较器

  • 写现代 C++ 时优先关注：默认初始化、RAII、避免裸指针、少用宏、少做不必要的手动资源管理
