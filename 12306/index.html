reduce query latency

  • SQL 查询重构
  • city_name → city_id 优化

把第一程、第二程、库存判断和结果排序放在同一层里一次性做，容易在中间阶段产生大量无效组合。

查询从“直接大 JOIN”改成了“分层”。

把整条查询拆成 first_leg、second_leg、transfer_candidates、available_candidates、ranked_candidates 五层，变成了“先缩小数据集，再 JOIN”

先把 first_leg 和 second_leg 各自过滤到较小的数据集，再做 JOIN，避免 ticket_price × ticket_price 先爆炸；

库存判断走 LEFT JOIN，避免相关子查询在每一行反复执行；

最后再统一排序和去重。
