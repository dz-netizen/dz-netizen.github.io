概述

数据库的概念

  • 数据库 database

存储和管理一组数据，通常按照某种数据模型来组织

  • 数据库系统 database management system

管理数据库的系统

  • 关系模型 relational model

关系模型中的实体和联系都可以用“表”来表示

  • 关系数据库 relational database

采用关系模型的数据库

  • 关系数据库系统 relational database management system

支持关系模型为核心模型的数据库系统

表（table/relation）

  • table/relation
  • 列（column）：一个属性，有明确的数据类型
    必须是原子类型，不能够再进行切割，没有内部结构

列数称之为度（degree），每一列是一个属性

  • 行（row）：一个记录（tuple，record）
    表是一个记录的集合，记录之间是无序的。

行数称之为基（cardinality）。

表的数学定义

K列的表：$\lbrace <t_1,…,t_k>|t1 \in D_1,…,t_k \in D_k \rbrace$

整个表是一个集合，$D_j$是第j列的类型所对应的所有可能取值的集合（域）。

Schema vs. Instanse

  • Schema:模式/类型

只需要定义一次，例如：
Student(ID integer,Name varchar(20),GPA float)

  • Instance: 实例/具体取值

具体存储哪些记录，每个列的具体值

关系模型

  • 优点：数据独立性

数据组织和存储的改变对于应用是隔离的

  • 每个属性是原子的，每行称为一个记录，行数称为基，列数称为度

ER模型（Entity Relationship model）实体联系模型

实体

  • 实体（Entity）
    现实世界中的对象，可以与其他对象区分

  • 实体集（Entity Set）
    相似实体形成的集合

  • 实体的属性（Attribute）
    一个实体集的每个实体都有的属性

erDiagram

    EntitySet{
        int attribute1 PK
        float attribute2
    }
  • 弱实体
    菱形加粗，实体框加粗，部分键下划线用虚线

实体集和属性的表示：

键

某些属性是特殊的，需要特别标记

  • 键key
    一个属性或者多个属性集合

唯一确定一个实体的属性集

实体集中两个不同的实体键不同

最小，多个属性组合，去除任何属性后都不是键

  • 候选键（candidate key）
    可能存在多个候选键

  • 主键（primary key）
    选定一个候选键为主键

  • 外键（foreign key）
    另一个表的主键
    唯一确定另外一个表的一个记录。

  • 主键的表示

主键包含的属性下加下划线

联系

如果联系是非堆成的，可以在边上标注联系中的角色。

实体集与联系集的关系

  • 实体参与一个联系，如果仅能参与一次，则需要加入一个箭头，即键约束（key constraint）
flowchart LR

    E1[EntitySet1]
    E2[EntitySet2]

    R{relationship}

    E1 ---> R
    E2 --- R

    A((attribute))
    R --- A
  • 如果可以多个联系，则不用箭头

粗线：每个实体都有联系

类层次

用一个倒三角（IS A）表示实体集之间的继承关系

类层次的约束

  • 交迭约束（overlap）
    两个子类之间是否可以有交集？存在共同的实体？
    默认：不相交（交集为空）
    如有，需要文字特殊说明

  • 覆盖约束（covering）
    多个子类的并集是否覆盖了父实体集？
    默认：没有覆盖约束（并集不是全集）
    如有，需要文字特殊说明

建模设计原则

  • 实体 vs. 属性
    考虑两方面，只有一个值还是多个值
    是简单的值，还是丰富的内部结构

关系代数

传统关系代数的关系运算

  • 代数
    用变量符号代替具体的数

  • 关系代数
    在关系上定义的代数

    “变量”：关系变量
    “数”：关系实例，即数据表
    “运算”、“函数”：关系运算

  • 传统关系代数
    将关系看作集合，传统关系代数的特征：没有重复的记录

  • 拓展关系代数
    关系可以看成多集（multi-set），允许重复的元素

选择、投影、连接

  • *选择 selectmax(*
    从一个表中提取一些行

关系代数形式：
$$\sigma_{conditions} (TableName)$$

  • 投影 projection
    从一个表中提取一些列

关系代数形式：
$$\pi_{\lbrace Row1Name,…,Row2Name \rbrace} (TableName)$$

在数据库系统的实现中，select默认不去重，用distinct去重。

  • 选择加投影

$$\pi_{\lbrace RowName \rbrace}(\sigma_{conditions}(TableName))$$

  • 连接join

已知两个表R和S，以R.a=S.b为条件的连接

$$R\bowtie_{R.a=S.b}S$$
R.a和S.b被称为join key。

  • 外部连接

left outer join :输出左表中没有匹配的记录
right outer join :输出右表中没有匹配的记录
full outer join :输出两表中没有匹配的记录

  • 非等值连接
    $\theta$ 连接
