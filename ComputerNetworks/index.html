电路交换

  • “交换” (switch) 的含义
    即转接——把一条电话线转接到另一条电话线，使它们连通起来。
    从通信资源的分配角度来看，“交换”就是按照某种方式动态地分配传输线路的资源

  • 特点
    面向连接
    必须经过三个步骤：
    建立连接（占用通信资源）、
    通话（一直占用通信资源）、
    释放连接（释放通信资源）

  • 以电路交换方式传送计算机数据效率低
    计算机数据具有突发性（burst），传送数据的时间不到10%，甚至低于1%
    面向连接的方式进行传输导致资源浪费，被占用的通信线路绝大部分时间空闲

分组交换

把较长的报文划分成较短的、固定长度的数据段

  • 每一个数据段前面添加上首部(header)构成分组(Packet)
    分组是互联网中传送的数据单元
    每个首部都包含地址等控制信息

  • 依次把各分组发送出去

分组交换原理

中间节点，以存储转发的形式将分组逐跳转发至目的端

数据通信的基本概念

常用术语

  • 信息：语音、文字、图像、视频等，通信目的即传送信息

  • 数据：运送信息的载体，有意义的符号序列

  • 信号：数据的电气或者电磁表现（
    模拟信号：代表信息的参数取值是连续的，
    数字信号：代表信息的参数取值是离散的）

  • 码元：在使用时域波形表示数字信号时，代表不同离散数值的基本波形

  • 单工通信
    发送器向接收器的单一固定方向上传输数据，没有反向交互
    例如无线电网络

  • 半双工通信
    双方通信设备既可以是发送器，也可以是接收器，某一时刻只能由一个方向向另外一个方向传输

  • 全双工通信
    双方通信设备既可以是发送器，也可以是接收器，某一时刻可以同时在两个方向上传输

  • 基带信号
    来自信源的原始信号，未经过调制（频谱搬移和变换），其特点是频率较低，信号频谱从零频附近开始，具有低通形式
    基带信号分为数字基带信号和模拟基带信号

  • 编码
    发送端吧二进制数据编码未数字信号，接收端再解码为比特

  • 调制
    使用载波，把基带信号的频率搬到高频，转换为模拟信号，称为带通调制/载波调制

固定编码

数字基带信号码型种类繁多，根据码元幅度取值不同可以分为

  • 二元码
    0，1

  • 三元码
    -1，0，1

  • 多元码
    工程上难以实现，信号间差距小，抗干扰能力弱

固定编码方式，存在大量直流分量，长串‘0’或者‘1’时，呈现固定连续电平，不出现跳变，无法提取时钟信号
极性反转，错误会传递下去

差分编码（0，1之间有关联）

利用相位差表示信息，不需要参考信号
仍存在直流分量，错误不会传导

信道容量与计算

  • 香农（shannon）

$C=W log_2^{1+S/N}$

其中C为信道容量（bps），W为信道带宽（Hz），S为信道平均功率，N为高斯白噪声功率，$\frac{S}{N}$为信噪比，常表示为SNR，$SNR=10 log_{10}^{S/N}$

-例子
SNR = 30dB,W=3000Hz
=> S/N = 1000
=> C = $3000 log_2^{1+1000}$ = 30kbps

差错检测

数据在传输过程中可能存在比特差错
（由于电磁干扰等原因）

  • 误码率$=\frac{error bits}{total bits}$

差错检测机制

基本思想：添加冗余信息

取得冗余信息和代价之间取得折中

  • 代价：计算代价/通信代价

当接收方检测到错误时，需要重传

flowchart LR
    A[send] --> |超时重传|B[receive]
    B --> |反馈|A

差错检测不能100%准确
系统近似100%

  • 实现方法：互联网每一层都做一次校验，进而可靠性越来越强
    即多层校验

  • 引入强秘密信息（密码学）

  • 单比特校验

  • 二维奇偶校验

每一行每一列都添加一个校验位
