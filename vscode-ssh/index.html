此种解决方法只需给对应连接设备加代理即可，有时实现不是很顺利（我是这样，呜呜呜），不多赘述

本地下载后上传

实现方式不是一成不变的，不同vscode及插件版本可能都会存在差异。

比如当容器是docker desktop，VS Code 可能是通过 WSL 再间接 attach 容器，一个版本的插件只需要我在WSL中安装了server就行。
后面插件版本更新后，发现不行了，新版 Dev Containers 插件更倾向于把 server 安装到容器内部。此时就需要将其置于容器内部

所需版本号、安装位置等都可以通过连接log得到，比如
test -d '/root/.vscode-server/bin/...
'Exit code 1
按照其要求安装好就行。
